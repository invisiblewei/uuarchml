# ChipViz Style Guide

**版本**: 0.1.0  
**日期**: 2026-02-26  

---

## 1. 设计哲学

### 1.1 核心原则

- **清晰优先**: 信息传递效率 > 视觉美观
- **减少认知负荷**: 一眼能看出数据流向和关键路径
- **工程师友好**: 符合芯片设计领域的视觉惯例

### 1.2 参考风格

参考经典计算机体系结构教材（如 *Computer Organization and Design*）中的数据通路图：

- 白色/浅色背景
- 清晰的阶段分隔
- 颜色编码功能类型
- 无多余装饰（阴影、渐变、圆角过多等）

## 3. 图元规范

### 3.1 block — 功能块

```
┌─────────────────┐
│     ALU         │  ← 标题栏（同色系深色）
├─────────────────┤
│                 │
│   (内容区域)     │
│                 │
└─────────────────┘
```

**尺寸**: 宽度 100-160px，高度 60-80px  
**圆角**: 4px  
**标题栏**: 高度 24px，填充色比主体深 10%

### 3.2 mux — 多路选择器

```
       OP1
      ╱   ╲
     ╱     ╲
    ╱_______╲
```

**渲染**: 梯形，上窄下宽  
**比例**: 上底 : 下底 : 高 = 0.6 : 1 : 0.5  
**标注**: 输入路数用小字标在下方

### 3.3 arbiter — 仲裁器

```
        Arb
         ◆
        ╱ ╲
       ╱   ╲
      ╱_____╲
```

**渲染**: 菱形（旋转45度的正方形）  
**尺寸**: 边长 50px  
**文字**: 水平显示（不随菱形旋转）

### 3.4 fifo — 队列

```
     ┌───────┐
    ╱   FB    ╲
   │   (4)    │
    ╲_________╱
```

**渲染**: 圆柱侧面投影  
**标注**: 深度用括号标在标签下方

### 3.5 port — 外部端口

```
┌═══════════════┐
║    IMem       ║
└═══════════════┘
```

**渲染**: 双边框（外框粗 2px，内框细 1px）  
**方向标记**: 
- master: 右侧小箭头 `→`
- slave: 左侧小箭头 `←`
- bidir: 双向箭头 `↔`

## 4. 布局规范

### 4.1 流水线阶段布局

```
┌──────────┬──────────┬──────────┬──────────┬──────────┐
│          │          │          │          │          │
│   IF     │   ID     │   EX     │   MEM    │   WB     │  ← 阶段标题
│          │          │          │          │          │
├──────────┼──────────┼──────────┼──────────┼──────────┤
│          │          │          │          │          │
│  [PC]    │ [Decode] │ [OP1   ] │ [DMEM]   │[Write   ]│  ← 模块
│    ↓     │    ↓     │  \   /   │    ↓     │ [Back]   │
│ [IMem]   │[Regfile] │   [ALU]  │          │    ↓     │
│          │          │    ↓     │          │[Regfile] │
├──────────┼──────────┼──────────┼──────────┼──────────┤
│  IF/ID   │  ID/EX   │  EX/MEM  │  MEM/WB  │          │  ← 流水线寄存器
└──────────┴──────────┴──────────┴──────────┴──────────┘
```

**参数**:
- 阶段宽度: 140-180px（根据内容自适应）
- 阶段间距: 0（共用边框）
- 模块间距: 16px
- 流水线寄存器高度: 24px

### 4.2 模块对齐

- 同阶段内模块**水平居中**
- 阶段标题**垂直居中**于标题栏
- 信号连接点位于模块**左右边缘中心**

### 4.3 信号路由

**基本原则**:
1. 从左到右的信号：直线或轻微弯曲
2. 从右到左的信号（旁路）：弧线或折线，从下方/上方绕行
3. 避免信号线交叉，必要时使用阶梯路由

```
正常流向:  ──────────────────▶

旁路流向:  ───────┐
                  │
         ◀────────┘
```

## 5. 字体规范

### 5.1 字体栈

```css
font-family: 'Segoe UI', 'SF Pro Display', -apple-system, BlinkMacSystemFont, sans-serif;
monospace: 'JetBrains Mono', 'Fira Code', 'Consolas', monospace;
```

### 5.2 字号

| 元素 | 字号 | 字重 |
|------|------|------|
| 阶段标题 | 14px | Bold (700) |
| 模块名称 | 13px | Semi-bold (600) |
| 信号标签 | 11px | Regular (400) |
| 位宽标注 | 10px | Regular (400) |
| 注释文字 | 12px | Regular (400) |
| 图例 | 11px | Regular (400) |

### 5.3 文字颜色

- 深色背景（阶段标题栏）: `#FFFFFF`
- 浅色背景（模块内部）: `#1A1A1A`
- 次要信息: `#666666`

## 6. 标注规范

### 6.1 流水线寄存器

```
┌─────────────────┐
│      IF/ID      │  ← 居中，白色文字，深色背景
└─────────────────┘
```

**背景**: `#37474F`  
**文字**: `#FFFFFF`  
**高度**: 24px

### 6.2 高亮区域

```
┌─────────────────────────┐
│  ╔═══════════════════╗  │
│  ║                   ║  │
│  ║   [OP1]  [OP2]    ║  │  ← 半透明背景覆盖
│  ║                   ║  │
│  ╚═══════════════════╝  │
│     3-to-1 Bypass       │  ← 标签在下方
└─────────────────────────┘
```

**边框**: 2px dashed，同色深 20%  
**背景**: 15% 透明度  
**标签**: 10px，大写，同色

### 6.3 关键路径

```
═══════════════════════════▶  ← 粗实线 (3px)
   CRITICAL PATH ~800ps       ← 红色标签
```

**线条**: 3px solid `#F44336`  
**发光效果**: 4px blur，`#F44336` 20% 透明度  
**标签**: 红色背景，白色文字

### 6.4 注释气泡

```
    ┌─────────────────┐
    │  Round-robin    │
    │  IF 优先级更高   │
    └────────┬────────┘
             │
            [Arb]
```

**气泡**: 浅黄背景 `#FFFDE7`，边框 `#F9A825`  
**连线**: 1px dashed `#999999`  
**箭头**: 指向目标模块中心

所有图表右下角应包含图例：

```
│  ─── Data  - - Control  ··· Bypass │
│  ═══ Critical Path                 │
└─────────────────────────────────────┘
```

## 8. 响应式规则

### 8.1 缩放适配

| 缩放级别 | 调整 |
|----------|------|
| 50% | 隐藏位宽标注，简化信号标签 |
| 75% | 隐藏次要信号 |
| 100% | 完整显示 |
| 150%+ | 显示详细注释 |

### 8.2 紧凑模式

当阶段数 > 6 时，自动切换紧凑模式：
- 模块高度减少 20%
- 字号减少 1px
- 隐藏非关键信号标签

## 9. 主题变体

### 9.1 Light（默认）

白底，彩色图元。适合文档、PPT。

### 9.2 Dark

深色背景，高对比度。适合长时间查看、演示。

```
背景:     #1A1A2E
图元填充: 比 Light 版深 20%
图元边框: 保持色相，提高亮度
文字:     #EAEAEA
```

### 9.3 Paper

米黄背景，仿手绘风格。适合打印、笔记。

```
背景:     #FEFCE8
网格:     #FDE68A (点状)
图元:     白色填充，黑色边框
```
