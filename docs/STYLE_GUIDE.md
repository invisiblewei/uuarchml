## 1. 设计哲学

### 1.1 核心原则

- **清晰优先**: 信息传递效率 > 视觉美观
- **减少认知负荷**: 一眼能看出数据流向和关键路径
- **工程师友好**: 符合芯片设计领域的视觉惯例


## 2. 色彩系统

### 2.1 主色调

```
背景:     #FFFFFF  (纯白)
网格线:   #E5E5E5  (浅灰)
主文字:   #1A1A1A  (近黑)
次文字:   #666666  (中灰)
```

### 2.2 图元颜色

| 图元 | 填充 | 边框 | 文字 |
|------|------|------|------|
| **block** | `#E8F4FD` | `#2196F3` | `#1565C0` |
| **mux** | `#F3E5F5` | `#9C27B0` | `#7B1FA2` |
| **arbiter** | `#FFF8E1` | `#FFC107` | `#FF8F00` |
| **fifo** | `#E8F5E9` | `#4CAF50` | `#2E7D32` |
| **port** | `#F5F5F5` | `#616161` | `#424242` |

### 2.3 信号颜色

| 类型 | 线条 | 说明 |
|------|------|------|
| **data** | `#2196F3` (蓝) | 数据通路 |
| **control** | `#FF9800` (橙) | 控制信号 |
| **clock** | `#9C27B0` (紫) | 时钟 |
| **reset** | `#757575` (灰) | 复位 |
| **bypass** | `#00BCD4` (青) | 旁路/前递 |
| **critical** | `#F44336` (红) | 关键路径 |

### 2.4 高亮颜色

| 用途 | 颜色 | 透明度 |
|------|------|--------|
| 区域高亮 (blue) | `#2196F3` | 15% |
| 区域高亮 (red) | `#F44336` | 15% |
| 区域高亮 (green) | `#4CAF50` | 15% |
| 区域高亮 (yellow) | `#FFC107` | 15% |
